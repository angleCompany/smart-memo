# Smart Memo v2.0 기획서
> 작성: 2026-05-14 | 방법: 기획자 2명 + 토론자 4명 멀티 에이전트 토론

---

## 제품 정의

> "macOS에서 링크를 던지는 마찰이 0인 앱. 정리 없이도 나중에 찾을 수 있다."

**포지셔닝**: Bear의 가벼움 + Raindrop의 URL 수집 + macOS 네이티브 속도
**핵심 차별화**: Capture-to-saved < 1초 / URL을 1급 객체로 취급 / 로컬 우선 + iCloud 옵션

## 확정 MVP — v2.0 (개발 순서)

| 순위 | 기능 | 복잡도 | 이유 |
|------|------|--------|------|
| **1** | Quick Capture 글로벌 단축키 (`Cmd+Shift+M`) | Medium | 핵심 가치 제안. 4명 전원 동의 |

## 기술 결정

| 항목 | 결정 | 근거 |
|------|------|------|
| **DB** | `better-sqlite3` | `node:sqlite`는 2026년 기준 미성숙 (동기 API only, 한국어 FTS 취약) |
| **iCloud** | 로컬 DB + iCloud Drive 백업 전용 | 직접 sync는 corruption 위험 (Bear·Drafts 초기 버전 모두 후회) |
| **동기화** | 단일 디바이스 우선, 동시 편집 미지원 명시 | v3.0에서 CloudKit 재검토 |
| **검색** | FTS5 BM25 | TF-IDF 자체 구현 금지 (한국어 품질 50%대) |
| **외부 deps** | "검증된 OSS 패키지 허용"으로 완화 | "Node 내장만" 제약은 비현실적 |
| **마이그레이션** | 스키마 버전 필드 + 자동 백업 후 마이그레이션 | 실패 시 v1 백업 파일로 롤백 |

## 절대 금지 목록 (Anti-patterns)

1. CRDT / 복잡한 동기화를 MVP에 넣기
2. 기능 5개 이상 동시 개발
3. 계정/로그인 강제
4. 블록 에디터 도입 (Notion화의 시작)
5. 수익 모델 없이 무기한 무료 유지
6. `node:sqlite` 프로덕션 채택 (2026 기준 미성숙)
7. iCloud Drive를 SQLite DB 직접 동기화 저장소로 사용
8. AI 기능 무계획 추가 (Readwise와 정면충돌)
9. 카테고리 + 태그 동시 제공 (결정 피로)
10. iOS 동시 개발 (v3.0 이후)
11. "파워유저용" 기능을 기본 UI에 노출
12. 썸네일 직접 첨부 (iCloud 용량 폭증, 저작권 회색지대)
13. 글로벌 단축키를 시스템/타 앱과 충돌하는 키로 기본 설정

## 기술 리스크

| 리스크 | 대응 |
|--------|------|
| macOS 접근성 권한 (globalShortcut) | 온보딩에서 권한 요청 UX 사전 설계 |
| URL 메타데이터 수집 실패 (SPA, Cloudflare) | timeout(5s) + 수동 입력 fallback UI |
| iCloud 로그아웃 시 데이터 접근 | 로컬 DB 항상 유지, iCloud는 미러만 |
| 단축키 충돌 (Raycast, Alfred) | 사용자 커스터마이징 UI 필수 |
| 한국어 FTS 검색 품질 | unicode61 토크나이저 + 분해 검색 보완 |
| 코드 사이닝 없는 Gatekeeper 경고 | xattr -cr 명령 안내 + 우클릭 열기 가이드 |

## 경쟁 차별화 KPI

- **캡처 속도**: 단축키 → 저장 완료 < 1초 (벤치마크 측정 의무화)
- **검색 응답**: 10만 개 메모에서 < 50ms
- **무계정 시작**: 설치 → 첫 저장 < 30초
- **온보딩**: 첫 화면에서 튜토리얼 슬라이드 0장
